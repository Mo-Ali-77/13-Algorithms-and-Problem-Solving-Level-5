# 🟢 Project 6: Stack Array Class

## 📝 Project Introduction
Continuing our journey into code reusability, this project involves building a **stack** data structure but with a different approach. 

Instead of relying on linked lists, we inherited/used the `Queue Array Class` (Project 5) to build our own stack. Just as we did in Project 3, we reused the same functions while modifying the behavior of the `Push` function to comply with the **LIFO** (Last In, First Out) principle, specifically by pushing elements to the beginning of the dynamic array.

💡 **Lessons Learned:**
Building multi-layered systems. The Stack relies on the Queue Array, which in turn relies on the Dynamic Array. This hierarchy reduces code duplication and centralizes maintenance.

# 🟢 المشروع السادس: فئة المكدس المعتمد على المصفوفة (Stack Array Class)

## 📝 مقدمة المشروع
استكمالاً لرحلة إعادة استخدام الكود (Code Reusability)، يمثل هذا المشروع بناء هيكل بيانات **المكدس (Stack)** ولكن بأساس مختلف. 

بدلاً من الاعتماد على القوائم المرتبطة، قمنا بوراثة/استخدام `Queue Array Class` (المشروع الخامس) لبناء الـ Stack الخاص بنا. تماماً كما فعلنا في المشروع الثالث، قمنا بإعادة استخدام نفس الدوال مع تعديل سلوك دالة الإضافة (Push) لتتوافق مع مبدأ **LIFO** (ما يدخل أخيراً يخرج أولاً) وتحديداً الإضافة في بداية المصفوفة الديناميكية.

💡 **الخبرة المكتسبة:**
بناء أنظمة متعددة الطبقات (Multi-layered Systems). الـ Stack يعتمد على الـ Queue Array، والذي بدوره يعتمد على الـ Dynamic Array. هذا التسلسل الهرمي يقلل من تكرار الكود ويجعل عملية الصيانة مركزية.
