# 🟢 Project 5: Queue Array Class

## 📝 Project Introduction
In the second project, we built the queue data structure using a “doubly linked list”. In this project, we completely changed the “internal engine”! 

We applied the principle of **composition** again, but this time we used the `Dynamic Array Class` (which we built in Project 4) as the underlying structure to run the Queue. 

💡 **Engineering Goal:**
This project demonstrates the power of **code architecture**. The external interface of the Queue (the functions used by the programmer, such as push and pop) remained the same, but the way it is stored in memory changed completely from separate nodes to an expandable block of contiguous memory.

## 🛠️ Concepts Applied 
* **Data Structure Swap:** Changing the underlying data structure without breaking the system.
* **Composition:** The `Queue Array` contains (HAS-A) a `Dynamic Array`.
* **Memory Layout:** Transitioning to contiguous memory storage, which improves cache locality.

# 🟢 المشروع الخامس: فئة الطابور المعتمد على المصفوفة (Queue Array Class)

## 📝 مقدمة المشروع
في المشروع الثاني، قمنا ببناء هيكل بيانات الطابور (Queue) بالاعتماد على "القائمة المرتبطة المزدوجة". في هذا المشروع، قمنا بتغيير "المحرك الداخلي" بالكامل! 

لقد قمنا بتطبيق مبدأ **التركيب (Composition)** مرة أخرى، ولكن هذه المرة استخدمنا `Dynamic Array Class` (الذي بنيناه في المشروع الرابع) كبنية تحتية لتشغيل الـ Queue. 

💡 **الهدف الهندسي:**
هذا المشروع يوضح قوة **التصميم المعماري للكود**. الواجهة الخارجية للـ Queue (الدوال التي يستخدمها المبرمج مثل push و pop) ظلت كما هي، ولكن طريقة التخزين في الذاكرة اختلفت تماماً من عقد متفرقة (Nodes) إلى كتلة متصلة من الذاكرة (Contiguous Memory) قابلة للتمدد.

## 🛠️ المفاهيم المطبقة 
* **Data Structure Swap:** تغيير البنية التحتية للبيانات دون كسر النظام.
* **Composition:** الـ `Queue Array` يحتوي على (HAS-A) `Dynamic Array`.
* **Memory Layout:** الانتقال إلى التخزين المتصل في الذاكرة مما يحسن من سرعة الوصول العشوائي (Cache Locality).
